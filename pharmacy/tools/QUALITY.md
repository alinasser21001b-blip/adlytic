# حوكمة جودة المستشار الصحي — صيدلية در الشارقة

## بوابات الجودة (تُشغَّل قبل كل نشر)

```bash
node pharmacy/tools/validate-kb.js   # تدقيق قاعدة المعرفة
node pharmacy/tools/test-engine.js   # اختبارات وحدات المحرك
```

أي فشل في أيٍّ منهما = لا نشر.

## ما يضمنه المدقّق (validate-kb)
- كل قاعدة/جرعة/حدّ أعلى/مانع استعمال يستشهد بمرجع موجود في سجلّ Layer 1.
- كل مرجع مكتمل: المنظمة، العنوان، النوع، السنة، الإصدار، تاريخ المراجعة،
  تاريخ آخر تحقق، مستوى الدليل، رابط https.
- لا «قواعد ميتة»: كل حقيقة تستخدمها القواعد لها سؤال يُنتجها.
- كل معرّف مغذٍّ/منتج مُشار إليه موجود فعلاً (الأهداف، الأسئلة، الكتالوج).
- وجود سؤالَي أمان إجباريين على الأقل وإخلاء مسؤولية.

## ما تضمنه اختبارات المحرك (test-engine) — 16 اختباراً
- **الأمان أولاً:** الإقصاء التام (هيموكروماتوز→حديد، كلى→مغنيسيوم/بوتاسيوم،
  فرط كالسيوم→كالسيوم/فيتامين د)؛ لا توصية أبداً وسؤال أمان معلّق، حتى عند
  بلوغ الحد الأقصى للأسئلة.
- **الجرعات المعيارية:** اختيار صف DRI الديموغرافي الصحيح (أنثى 19–50: حديد
  18 ملغ؛ فوق 70: فيتامين د 800 وحدة؛ الحامل: صف 27 ملغ).
- **التسلسل:** الهدف أولاً، الأمان قبل الأعراض، تخطّي سؤال الحمل للذكور.
- **الثقة:** محصورة [0,1]، المتصدر منطقي، ≤3 توصيات فوق العتبة.
- **الشفافية:** كل توصية تحمل أثر شرح + فحص أمان + مرجعاً أساسياً صالحاً.

## دورة المراجعة العلمية
1. **نصف سنوياً** (أو عند صدور تحديث من WHO/NIH/EFSA/IOM): راجع كل مدخل في
   `references` — حدّث year/version/url إن تغيّر المصدر، وحدّث
   `lastVerified` بتاريخ التحقق حتى لو لم يتغيّر شيء.
2. أي تعديل قيمة جرعة يجب أن يطابق حرفياً الجدول الرسمي للمصدر المستشهد به.
3. ارفع `meta.version` عند أي تغيير محتوى، وسجّل التغيير في رسالة الـcommit.
4. شغّل بوابتي الجودة قبل الدفع.

## قواعد لا تُكسر
- المحرك (`advisor.js`) لا يحتوي أي معرفة طبية ولا يخترع أي رقم.
- التوسيع يتم في `advisor-kb.js` فقط.
- لغة العرض تثقيفية دائماً («قد يناسب مناقشته مع مختص») — لا تشخيص ولا وصف.
- النهاية دائماً تسليم لصيدلي حقيقي عبر واتساب.
- التحليلات مجهولة تماماً — لا اسم/هاتف/نص حر.
