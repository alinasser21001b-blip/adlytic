المشكلة: 3 خدمات تبني من نفس GitHub → طابور Queued لساعات.
الحل:  adlytic فقط ينشر من GitHub. الباقي بدون auto-deploy.

─── الخطوة 1: ألغِ كل Queued ───
  • adlytic / thriving-manifestation / lively-insight
  • Deployments → Cancel (كل ما هو Queued أو Building)

─── الخطوة 2: thriving-manifestation ───
  Settings → Source → Auto Deploy: DISABLE
  Settings → Config File Path: /railway.sync-worker.json
  Variables (تأكد): SERVICE_ROLE=worker

─── الخطوة 3: lively-insight ───
  Settings → Source → Auto Deploy: DISABLE
  Settings → Config File Path: /railway.worker.json

─── الخطوة 4: adlytic (الإنتاج) ───
  Settings → Config File Path: /railway.json
  Settings → Source → Auto Deploy: ENABLE (branch: main)
  Public URL: adlytic-production.up.railway.app

─── الخطوة 5: انشر adlytic ───
  adlytic → Cmd+K → Deploy Latest Commit
  انتظر 5–7 دقائق (خدمة واحدة فقط)

─── ملاحظة: Publishing image بطيء؟ ───
  المشروع يستخدم Dockerfile (Alpine) بدل Nixpacks — صورة ~150MB بدل ~387MB.
  إذا علّق على Publishing image >10 دقائق: Cancel → Deploy Latest Commit.

─── تحقق ───
  curl https://adlytic-production.up.railway.app/api/health
  Hard refresh: /recommendations (لا خطأ escHtml)

─── (اختياري) نشر من GitHub Actions ───
  1. Token من https://railway.app/account/tokens (No Team)
  2. GitHub → Secrets → RAILWAY_TOKEN
  3. ثم يمكن تعطيل Auto Deploy على adlytic والاعتماد على Action فقط
